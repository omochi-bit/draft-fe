# TODO管理 Web アプリケーション：PRD（Product Requirements Document）

## 1. プロダクト概要

### 1.1 目的
ユーザーがタスク管理を効率的に行える Web アプリケーションを提供する。  
タグによる分類を可能とし、ユーザーごとに独立したデータ領域を持つ。

### 1.2 ターゲットユーザー
- 個人学習者
- 生産性向上を目的とした一般ユーザー
- 小規模プロジェクトの個人利用

### 1.3 プラットフォーム
- Webアプリケーション
- PC / モバイルブラウザ対応

---

## 2. 主要機能（MVP）

### 2.1 ユーザー認証
- メールアドレス＋パスワードでサインアップ
- ログイン / ログアウト
- パスワードリセット（任意）
- 認証済ユーザーのみアプリにアクセス可能
- 各ユーザーのデータは完全に分離

### 2.2 タスク管理機能

#### 2.2.1 タスク一覧表示
- ログインユーザーのタスクのみ表示
- 表示項目: タイトル, ステータス（未完了 / 完了）, 作成日時, 紐づくタグ（複数）
- フィルタ: 未完了 / 完了, タグフィルタ（指定タグに紐づくタスクのみ表示）
- 並び順: 作成日降順（デフォルト）

#### 2.2.2 タスク登録
- 入力項目: タイトル（必須）, 説明（任意）, タグ（複数選択可）
- 登録後は一覧へ即時反映

#### 2.2.3 タスク詳細
- 表示項目: タイトル, 説明, ステータス, タグ, 作成日時, 更新日時

#### 2.2.4 タスク編集
- 編集可能項目: タイトル, 説明, ステータス, タグ（複数）

#### 2.2.5 タスク削除
- 削除確認ダイアログの表示
- 削除後に一覧から除去

#### 2.2.6 完了フラグ変更
- 一覧からワンクリックで「未完了 ⇄ 完了」に切り替え可能

### 2.3 タグ管理機能

#### 2.3.1 タグ一覧表示
- ユーザーが作成したタグ一覧を表示
- 表示項目: タグ名, 作成日時

#### 2.3.2 タグ作成
- 入力項目: タグ名（必須／ユニーク）

#### 2.3.3 タグ編集
- タグ名変更（重複不可）

#### 2.3.4 タグ削除
- タグを削除すると Task と Tag の紐づき関係も削除される
- タスクそのものは削除されない

---

## 3. 非機能要件

### 3.1 セキュリティ
- 認証必須
- パスワードはハッシュ化して保存
- 各ユーザーのタスク・タグは分離
- CSRF / XSS / インジェクション対策

### 3.2 パフォーマンス
- 一般的操作は 500ms 以内を目標

### 3.3 拡張性
- プロジェクト機能
- タグカラー管理
- カレンダー機能
- 通知機能などを追加可能な構造とする
